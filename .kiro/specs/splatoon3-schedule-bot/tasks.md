# Implementation Plan

- [x] 1. プロジェクト基盤セットアップ
- [x] 1.1 TypeScriptプロジェクトの初期設定
  - TypeScriptのビルド設定（strictモード有効化、出力先ディレクトリ設定）を構成する
  - プロジェクトのビルドスクリプトと起動スクリプトを設定する
  - `.env`ファイルのテンプレートを作成し、必要な環境変数（`DISCORD_TOKEN`, `DISCORD_APPLICATION_ID`, `DATABASE_URL`）を定義する
  - _Requirements: 6.5_

- [x] 1.2 環境変数の読み込みとバリデーション
  - Zodスキーマを使って環境変数をバリデーションし、型安全な設定オブジェクトを返す仕組みを実装する
  - Discord Botトークン、アプリケーションID、データベースURLが未設定または不正な場合、起動時にエラーメッセージを表示して終了する
  - _Requirements: 6.3_

- [x] 1.3 Prismaセットアップとデータベーススキーマ定義
  - Prisma 7をインストールし、SQLite（better-sqlite3アダプター）用のスキーマを定義する
  - 通知設定テーブル（ギルドID、チャンネルID、通知時刻）をモデルとして定義し、ギルドID＋チャンネルIDのユニーク制約を設定する
  - 初回マイグレーションを作成し、Prisma Clientを生成する
  - _Requirements: 1.5_

- [x] 2. Spla3 APIクライアント
- [x] 2.1 (P) 対戦スケジュール取得機能
  - Spla3 APIの一括取得エンドポイントからレギュラー・バンカラ（チャレンジ/オープン）・X・イベント・フェスの対戦スケジュールデータを取得する
  - APIレスポンスのスネークケースフィールドをキャメルケースに変換する
  - リクエスト時にUser-Agentヘッダーを設定する
  - Zodスキーマで全フィールドをバリデーションし、スキーマ不一致時はエラーをログに記録してスローする
  - _Requirements: 5.1, 5.2, 5.4, 5.5, 5.6_

- [x] 2.2 (P) サーモンランスケジュール取得機能
  - Spla3 APIの専用エンドポイントからサーモンランのスケジュールデータ（ステージ、ブキ、ボス情報）を取得する
  - リクエスト時にUser-Agentヘッダーを設定する
  - Zodスキーマでレスポンスをバリデーションし、スキーマ不一致時はエラーをログに記録してスローする
  - _Requirements: 5.1, 5.3, 5.4, 5.5, 5.6_

- [x] 3. 通知設定の永続化
- [x] 3.1 通知設定のCRUD操作
  - Prisma Clientを使って通知設定の登録（upsert）、削除、時刻による検索、ギルド＋チャンネルによる検索を実装する
  - 登録時は同一ギルド・チャンネルの既存レコードがあれば通知時刻を上書き更新する
  - 削除時は該当レコードが存在しない場合にfalseを返す
  - _Requirements: 1.1, 1.4, 1.5, 2.1, 2.3_

- [x] 4. Discord Bot基盤
- [x] 4.1 Botクライアントの初期化と接続
  - discord.jsを使ってDiscord Botクライアントを作成し、ゲートウェイに接続する
  - 接続完了時にログメッセージを出力する
  - 接続断時の自動再接続はdiscord.jsの既定動作に委ねる
  - _Requirements: 6.1, 6.2, 6.4_

- [x] 4.2 スラッシュコマンドの登録
  - `/subscribe`（チャンネルと通知時刻を引数に持つ）と`/unsubscribe`（チャンネルを引数に持つ）のトップレベルコマンドを定義する
  - 通知時刻は整数型（0〜23）のオプションとして定義する
  - Bot起動時にコマンドをDiscordに登録する
  - _Requirements: 4.1, 4.2, 4.3, 6.2_

- [x] 5. コマンドハンドラー
- [x] 5.1 通知登録コマンドの処理
  - ユーザーが`/subscribe`を実行した時に、指定チャンネルと通知時刻を通知設定として保存する
  - 同一チャンネルが既に登録されている場合は通知時刻を上書き更新し、更新された旨をEmbedで表示する
  - 新規登録時は登録完了メッセージ（チャンネル名と通知時刻を含む）をEmbedで表示する
  - DB操作でエラーが発生した場合はエラーメッセージをユーザーに返す
  - _Requirements: 1.1, 1.2, 1.3, 1.4_

- [x] 5.2 通知解除コマンドの処理
  - ユーザーが`/unsubscribe`を実行した時に、指定チャンネルの通知設定を削除する
  - 削除完了時はEmbedで解除完了メッセージを表示する
  - 指定チャンネルが登録されていない場合は、登録が存在しない旨をEmbedで表示する
  - _Requirements: 2.1, 2.2, 2.3_

- [x] 6. 通知サービスとEmbed生成
- [x] 6.1 スケジュール通知メッセージの生成
  - 対戦スケジュールとサーモンランスケジュールの両方を取得し、全種別をまとめた通知メッセージをDiscord Embed形式で生成する
  - 各スケジュール種別（レギュラー、バンカラチャレンジ、バンカラオープン、X、イベント、サーモンラン、フェス）ごとにステージ名・ルール名・開始/終了時刻を見やすく整形する
  - イベントマッチはイベント名と説明も含める。開催中・予定がない場合はスキップする
  - サーモンランはステージ名・支給ブキ・ボス情報を含める
  - フェスは開催中・予定がある場合のみ表示する
  - _Requirements: 3.2, 3.3_

- [x] 6.2 通知の送信とエラーハンドリング
  - 指定されたチャンネルIDに対してEmbed群を送信する機能を実装する
  - Spla3 APIからのデータ取得に失敗した場合は、通知チャンネルにエラーメッセージを送信する
  - チャンネルが削除済みや権限不足で送信できない場合はログに記録してスキップする
  - _Requirements: 3.2, 3.4_

- [x] 7. 定期実行スケジューラー
- [x] 7.1 定期通知ジョブの実装
  - node-cronで毎時0分に実行されるジョブを登録する
  - 実行時に現在の時刻（時）でDBから該当する通知設定を検索し、各チャンネルに通知を送信する
  - Bot起動時にジョブを開始し、シャットダウン時にジョブを停止する
  - _Requirements: 3.1_

- [x] 8. アプリケーション統合とエントリポイント
- [x] 8.1 メインエントリポイントの実装
  - 環境変数読み込み → Prisma Client初期化 → Discord Bot接続 → スラッシュコマンド登録 → スケジューラー起動の順で各コンポーネントを結合する
  - シグナル（SIGINT/SIGTERM）を受け取った時にBot接続の切断とPrisma Clientの切断をグレースフルに行う
  - 全コンポーネントが正常に起動したことをログに出力する
  - _Requirements: 6.1, 6.2, 6.3_

- [x] 9. Docker構成
- [x] 9.1 Dockerfileの作成
  - マルチステージビルドでTypeScriptのビルドとPrisma Client生成をビルドステージで行い、本番ステージには実行に必要なファイルのみコピーする
  - コンテナ起動時にPrismaマイグレーション（deploy）を実行してからアプリケーションを起動するエントリポイントスクリプトを作成する
  - _Requirements: 6.5_

- [x] 9.2 docker-compose設定
  - Botサービスを定義し、SQLiteファイル用のNamed Volumeを設定してデータを永続化する
  - `.env`ファイルから環境変数を読み込む設定を行う
  - コンテナの自動再起動（unless-stopped）を設定する
  - _Requirements: 1.5_

- [x] 10. Terraformインフラ構成
- [x] 10.1 (P) OCIネットワークリソースの定義
  - VCN、パブリックサブネット、Internet Gateway、Route Table、Security List（SSH用イングレスルール）をTerraformで定義する
  - 入力変数としてコンパートメントID、リージョン等を外部化する
  - _Requirements: 6.1_

- [x] 10.2 (P) Compute Instanceの定義
  - ARM Ampere A1シェイプ（1 OCPU / 6 GB RAM）のCompute Instanceを定義する
  - cloud-initスクリプトでDocker EngineとDocker Composeを自動インストールする設定を含める
  - SSH公開鍵の設定と出力値（インスタンスのパブリックIP）を定義する
  - _Requirements: 6.1_
