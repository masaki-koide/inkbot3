# Implementation Plan

- [x] 1. subscribe コマンドのオプション簡略化
- [x] 1.1 subscribe コマンド定義から channel オプションを削除し、hour オプションを任意化する
  - コマンド定義から channel オプション（addChannelOption）を完全に削除する
  - hour オプションを必須から任意に変更する（デフォルト値は設定しない。バリデーション 0〜23 は維持）
  - import 文から不要になった `ChannelType` を削除する
  - _Requirements: 1.2, 3.3_

- [x] 1.2 subscribe ハンドラで実行チャンネルを使用し、hour にデフォルト値を適用する
  - チャンネル ID の取得方法を、オプションからの取得から、コマンド実行チャンネルの ID 取得に変更する
  - hour の取得を任意オプションとして扱い、省略時はデフォルト値 7 を適用する
  - 応答メッセージにチャンネルと通知時刻を表示する処理を維持する（変数名の調整のみ）
  - サーバー外実行時のエラーハンドリングは現行通り維持する
  - _Requirements: 1.1, 1.3, 3.1, 3.2, 4.1_

- [x] 2. unsubscribe コマンドのオプション簡略化
- [x] 2.1 unsubscribe コマンド定義から channel オプションを削除する
  - コマンド定義から channel オプション（addChannelOption）を完全に削除する
  - _Requirements: 2.2_

- [x] 2.2 unsubscribe ハンドラで実行チャンネルを使用する
  - チャンネル ID の取得方法を、オプションからの取得から、コマンド実行チャンネルの ID 取得に変更する
  - 応答メッセージにチャンネルを表示する処理を維持する（変数名の調整のみ）
  - 登録が存在しない場合の警告メッセージは現行通り維持する
  - サーバー外実行時のエラーハンドリングは現行通り維持する
  - _Requirements: 2.1, 2.3, 4.2_

- [x] 3. テスト
- [x] 3.1 コマンドハンドラのテストを追加する
  - subscribe ハンドラが実行チャンネル ID を通知先として使用することを検証する
  - subscribe ハンドラが hour オプション省略時にデフォルト値 7 で登録することを検証する
  - subscribe ハンドラが hour オプション指定時にその値で登録することを検証する
  - unsubscribe ハンドラが実行チャンネル ID を解除対象として使用することを検証する
  - _Requirements: 1.1, 2.1, 3.1, 3.2_
